Class StringLength

A rule that passes is a validated string is between an inclusive minimum/maximum length.

Remarks

Either of Min or Max may be null, in which case they are ignored and not used. If either are null then this rule becomes effectively "shorter than" or "longer than". If both are null then this rull will always pass.

If the string itself is null then this rule will pass. To prevent null strings, combine this rule with NotNull.

The logic of this rule does not verify that the Mininum length is not greater-than the Maximum length, or that neither of these values is negative. Thus it is possible to set up scenarios where this rule will always return a failure result for any non-null string, as the pass criteria cannot be satisfied.

This rule will always return a synchronous result.
