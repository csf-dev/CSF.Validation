Interface IRule<TValidated>

A service for a validation rule which validates an instance of a specified object type.

Namespace: CSF.Validation.Rules
Assembly: CSF.Validation.Abstractions.dll
Syntax
public interface IRule<in TValidated>
Type Parameters
Name Description
TValidated

The type of the object which this rule validates
Remarks

A validation rule should be a single piece of logic which tests a single value against a business rule/requirement to determine whether it is valid according to that business rule or not. Whilst it is possible to write dependencies between validation rules (EG: Don't run this rule if that other rule failed), validation rules should aim to be as independent as possible.

The generic type of this interface is contravariant. That means that (for example) an IRule<Animal> may be used as if it were an IRule<Dog>.

You are encouraged to read more at the documentation for writing rule classes & best practices for writing rules.
